  2. Option A (Free & easy): Deploy to GitHub Pages by creating a repository named username.github.io, commit this file to main, then enable Pages from the repository settings.
  3. Option B (drag-and-drop + custom domain): Use Netlify — simply drag & drop the site folder or connect a Git repo. Configure custom domain & SSL in Netlify dashboard.
  4. Option C (fast CI, previews): Use Vercel — import the Git repo and set domain settings; automatic deployments for every push.
  5. Option D (scalable, enterprise): Host on AWS S3 + CloudFront and use Route 53 for DNS + AWS Certificate Manager for SSL.
  6. DNS & custom domain: Buy a domain from any registrar (GoDaddy, Google Domains, Namecheap). Then point A/CNAME or use the provider instructions to connect to GitHub/Netlify/Vercel. Update DNS TTL and wait (usually minutes to a few hours) for propagation.
  7. Verify SSL: GitHub Pages, Netlify and Vercel automatically issue Let's Encrypt certificates for custom domains. For AWS CloudFront, request a certificate with AWS Certificate Manager in the same region/edge configuration.
